# Паспорт проекта: ESG Data Inventory

## 1. Название
ESG Data Inventory (Университетский каталог ESG-данных)

## 2. Описание проекта (2-4 предложения)
Проект представляет собой веб-систему для централизованного учета, импорта и управления метриками ESG (экология, социальная ответственность, управление) в университете. Позволяет хранить информацию о метриках, источниках данных, подразделениях и хранилищах, а также отслеживать статус сбора и качество данных. В системе реализована роль-based авторизация, гибкий импорт данных (Google Sheets, CSV, JSON, ручной ввод) и логирование аудит-транзакций.

## 3. Цель
Создать унифицированный инструмент учета ESG-метрик в университете для повышения прозрачности, качества данных и скорости подготовки отчетности по устойчивому развитию.

## 4. Задачи
- Реализовать безопасную аутентификацию (JWT) и ролевую модель (admin/editor/viewer).
- Разработать импортер данных из нескольких источников (Google Sheets, CSV, JSON).
- Сформировать модель импорта с пакетной трассировкой (`ImportBatch`) и сохранением сущности как одной метрики.
- Обеспечить интерфейс управления метриками, источниками, подразделениями и хранилищами.
- Добавить многоязычность (ru/kk/en), документацию и возможность деплоя (Docker/Render/Netlify).

## 5. Актуальность/проблема
В вузах часто данные ESG разбросаны по разным системам, отделам и файлам, из-за чего сложна подготовка отчетности и анализ. Проект решает эту проблему, создавая единый каталог, в котором метрики и источники данных централизованно хранятся и отслеживаются.

## 6. Решение
Разработана веб-платформа, включающая:
- backend `Node.js + Express + Prisma + PostgreSQL`; 
- frontend `Vue 3 + Vite`; 
- email-сервис с Nodemailer для сброса пароля;
- работу с импортуемыми файлами как с целыми метриками (без разбивки по строкам);
- аудиторский лог изменений и CRUD-интерфейс.

## 7. Инструменты и технологии
- Node.js 20+, Express
- PostgreSQL + Prisma ORM
- Vue 3, Pinia, Vue I18n, Axios
- Jest для backend тестов
- Docker, docker-compose
- Netlify frontend, Render backend
- SMTP / Nodemailer

## 8. Результат
- Рабочий продукт с интерфейсом импорта, управления и фильтрации метрик.
- Полностью локализованная страница импорта на трех языках.
- 100% пройденные тесты (22/22).
- Устранена проблема "дробления файла", метрика сохраняется как единый объект.
